# Ticket Resolution AI - React Frontend

A modern, responsive React UI for the Ticket Resolution AI platform. Built with React 18 and styled with custom CSS.

## 📁 Project Structure

```

├── public/
│   └── index.html          # Main HTML file

│   │   ├── HealthStatus.jsx

│   │   ├── IssueResult.jsx
│   │   └── IssueResult.css
│   ├── App.jsx            # Main app component
│   ├── App.css            # App styling
```

## 🚀 Getting Started

### Prerequisites
- Node.js (v14 or higher)
- npm (v6 or higher)
- Backend API running on `http://localhost:8000`

### Installation

1. Navigate to the frontend folder:
```bash
cd frontend
```

2. Install dependencies:
```bash
npm install
```

### Development

Start the development server:
```bash
npm start
```

The app will open at `http://localhost:3000`

### Build

Create an optimized production build:
```bash
npm run build
```

## 🎨 Features

### Dashboard Tab
- **System Overview**: Real-time system health status
- **Quick Stats**: Total tickets, resolved issues, pending assignments
- **Recent Tickets**: List of the latest processed tickets
- **Click to View**: Click any recent ticket to see detailed results

### Create Ticket Tab
- **Issue Description**: Rich text area for detailed issue description
- **Severity Level**: Select from Low, Medium, or High priority
- **AI Processing**: Submit for automatic analysis and resolution
- **Feedback**: Real-time success/error messages
- **Examples & Tips**: Helpful hints for writing good issue descriptions

### Results Tab
- **Root Cause Analysis**: AI-identified root cause with confidence score
- **Remediation Steps**: Step-by-step solutions generated by AI
- **Assignment**: Developer assignment for unresolved issues
- **Execution Metrics**: Processing time and status for each workflow step
- **Timeline**: Creation and update timestamps
- **Processing Log**: Detailed log of all processing steps

### Health Status
- **Live Indicator**: Real-time system status with pulsing indicator
- **Refresh Button**: Manual health check button
- **Workflow Status**: Shows if AI workflow is ready

## 🔌 API Integration

The frontend communicates with the backend API at `http://localhost:8000`:

### Endpoints Used
- `GET /health` - Check system health
- `POST /issues` - Create and process a new ticket
- `GET /issues/{issue_id}` - Retrieve ticket details

### CORS Configuration
The frontend proxies requests through `proxy: "http://localhost:8000"` in `package.json`, so all API calls go to the backend automatically.

### IssueResult.jsx

- Health text

## 🐛 Troubleshooting

### API Connection Issues
- Ensure backend is running on `http://localhost:8000`
- Check browser console for CORS errors
- Verify the proxy setting in `package.json`

### React Warnings
- Warnings about unused variables should be fixed before production build
- StrictMode enables additional checks in development

### Building for Production
```bash
npm run build
```
The optimized build will be in the `build/` folder.
